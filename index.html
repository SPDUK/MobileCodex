The Support Layer Codex Mobile Deserves

Codex Mobile gives you the control surface. MobileCodex adds the operational layer that makes agent work easy to trust when you are away from the laptop.

Readiness Check

Ask: "Doctor this workspace." It checks runtimes, ngrok, UX proof, proof storage, registry health, and skill drift.

Session Snapshot

Ask: "Snapshot this before I step away." It returns branch, dirty files, ports, servers, recent proof, and next refs.

Tracked Previews

Ask: "Start the app preview." Codex tracks command, PID, port, local URL, running status, and log path.

Phone Preview URLs

Ask: "Give me a safe phone URL." It verifies locally, gates sensitive apps, then extracts the real ngrok URL.

Command Proof

Ask: "Run the checks and show only what matters." You get exit codes, key output, artifacts, and blockers.

Live Server Status

Ask: "What previews are still running?" It lists MobileCodex-owned servers and avoids stopping unknown processes.

Proof Bundles

Ask: "Bundle the proof." Screenshots, logs, changed files, server state, and snapshot details land in one file.

UX Proof

Ask: "Capture mobile proof for this page." It saves phone and desktop screenshots with browser evidence when available.

Designed Around The Codex Mobile Handoff

MobileCodex turns short phone prompts into concrete agent workflows. The user asks in plain English; Codex gets a repeatable command, proof target, and handoff shape.

"What Is Going On?"

Runs a session snapshot so the phone view shows branch, changed files, ports, servers, recent proof, and next refs.

"Keep The Preview Running."

Starts a registered server with PID, port, URL, command, and log file so the preview can be resumed or stopped later.

"Show Me It Works."

Captures mobile and desktop screenshots, summarizes browser status, and reports exact blockers instead of vague confidence.

"Package The Proof."

Creates a proof bundle that can be reviewed from chat after the user leaves the laptop or switches sessions.

Add It To Your Codex Mobile Workflow

Install MobileCodex beside your existing Codex setup. It does not replace the official mobile experience; it gives Codex better defaults when that experience becomes your main control surface.

1

Open Official Codex Mobile

Open chatgpt.com/codex/mobile to continue the Codex session that already has your repo, tools, browser, and local workspace.

2

Install The Support Skill

Place the skill folder under your Codex skills directory so Codex can discover the mobile support workflow.

3

Run Doctor

Check ngrok, UX proof, server registry health, proof storage, skill files, and installed-copy drift before you need them.

4

Ask From Your Phone

Say "Snapshot this", "Start a preview", "Capture UX proof", or "Bundle the proof" and get phone-readable output.

What Users Actually Feel

MobileCodex removes the awkward parts of mobile agent work: invisible terminals, stale localhost links, noisy logs, and unclear stopping points.

No Lost Context

Before you step away, Codex can snapshot branch, changed files, ports, servers, proof, and next steps.

No Mystery Localhost

Previews come back with tracked server status, local URL, optional public URL, and exact blocker if tunneling is unavailable.

No "Looks Good"

No Fuzzy Endings

Every handoff can include result, preview, proof, current state, and the safest next action.
